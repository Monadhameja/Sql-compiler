I am responsible for designing and implementing the lexical analysis phase of the SQL Query Compiler. In this phase, I developed a tokenizer that breaks down the input SQL query into meaningful tokens such as keywords (SELECT, WHERE), identifiers (column names), operators (=, >, <), and values (strings and numbers). I used regular expressions to accurately identify and classify different token types while ensuring that the system can handle both valid and invalid queries. Even in cases of incorrect syntax, I ensured that tokens are still generated to support error detection and debugging. Additionally, I worked on improving the visualization of tokens in the user interface so that users can clearly understand how their query is processed internally. My contribution helped in building a strong foundation for the parsing and execution phases of the compiler.
